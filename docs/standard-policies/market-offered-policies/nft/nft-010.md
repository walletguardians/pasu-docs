# NFT-010: 컬렉션을 통째로 내주는 오퍼에 서명할 시 차단

### Policy Definition (정책 정의)

> Seaport 주문에서 **특정 NFT 한 점이 아니라 컬렉션 전체**를 내주는 오퍼에 서명하려 하면 차단합니다.

Seaport에서 오퍼(Offer)에 서명하면 내가 내놓는 NFT와 받을 대금이 한 묶음으로 정해집니다. 보통은 토큰 ID로 어떤 NFT를 줄지 한 점씩 콕 집어 적습니다. 그런데 토큰 ID 대신 "이 컬렉션에 속한 것이면 무엇이든"이라는 조건(criteria)으로 오퍼를 만들면, 상대는 내가 일일이 고르지 않은 NFT까지 그 조건에 맞는 대로 가져갈 수 있습니다. 이런 모양의 오퍼는 사람이 직접 만들 일이 거의 없고, 지갑을 터는 서명을 그럴듯하게 포장한 경우가 대부분입니다. 따라서 오퍼가 토큰 ID가 아닌 컬렉션 조건으로 NFT를 내주는 형태이면, 서명 자체를 막습니다.

#### Scope (적용 범위)

Seaport 주문 서명에 적용됩니다. 내가 내놓는 오퍼가 특정 토큰 ID가 아니라 컬렉션 조건(criteria)으로 NFT를 지정할 때입니다. (Sign Order)

#### Audience (대상 사용자)

OpenSea 등 Seaport 마켓플레이스에서 NFT를 거래하는 사용자

#### Used Data (판정에 사용될 데이터)

주문이 오가는 마켓플레이스(`context.venue.name` — seaport)와, 내가 내놓는 오퍼가 토큰 ID 대신 컬렉션 조건으로 지정됐는지 여부(`context.offerHasCriteria`)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("offer-collection-giveaway-deny")
@severity("deny")
@reason("서명하면 구매자가 이 컬렉션에서 본인이 일일이 고르지 않은 NFT까지 가져갈 수 있습니다 — 한 점이 아니라 컬렉션 전체를 내주는 서명이라 차단했습니다")
forbid (
    principal,
    action == Marketplace::Action::"SignOrder",
    resource
)
when
{
    context.venue.name == "seaport" &&
    context.offerHasCriteria == true
};
```
