# NFT-003: NFT를 소각 주소로 보낼 시 차단

### Policy Definition (정책 정의)

> NFT를 **소각 주소**로 보내는 전송을, 자산이 사라지기 전에 **차단**합니다.

소각 주소는 개인키(Private Key)를 아무도 알 수 없는 0x000…000·0x000…dead 같은 주소입니다. 여기로 보낸 자산은 누구도 다시 꺼낼 수 없습니다. NFT는 토큰과 달리 같은 것이 하나뿐이라, 한 번 소각 주소로 넘어가면 그 NFT 자체를 영영 되찾을 수 없습니다. 그래서 NFT를 소각 주소로 보내려 하면 경고에 그치지 않고 전송 자체를 막습니다.

#### Scope (적용 범위)

NFT 전송(NftTransfer)에 적용됩니다. (받는 주소가 0x000…000 또는 0x000…dead일 때)

#### Audience (대상 사용자)

NFT를 보유하거나 거래하는 모든 사용자

#### Used Data (판정에 사용될 데이터)

NFT를 받을 주소(`context.recipient`)가 소각 주소(0x000…000 / 0x000…dead) 목록에 들어 있는지

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("nft-transfer-burn-recipient-deny")
@severity("deny")
@reason("이 NFT를 소각 주소로 보내는 거래입니다. 한 번 보내면 영영 되찾을 수 없어 차단합니다.")
forbid(principal, action == Token::Action::"NftTransfer", resource)
when {
  ["0x0000000000000000000000000000000000000000",
   "0x000000000000000000000000000000000000dead"].contains(context.recipient)
};
```
{% endcode %}
