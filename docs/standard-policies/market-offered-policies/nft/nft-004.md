# NFT-004: 블록리스트에 오른 주소로 NFT를 전송할 시 차단

### Policy Definition (정책 정의)

> 받는 주소가 알려진 드레이너·제재 **블록리스트**에 올라 있으면, NFT 전송이 나가지 않도록 **차단**합니다.

NFT를 전송하면 소유권이 받는 주소로 그대로 넘어가고, 한 번 나간 전송은 되돌릴 수 없습니다. 받는 주소가 지갑을 털어온 드레이너 주소이거나 제재 목록에 오른 주소라면, NFT를 보내는 순간 그대로 빼앗기거나 회수 길이 막힙니다. 그래서 받는 주소가 블록리스트에 올라 있으면 경고에 그치지 않고 전송 자체를 막습니다.

#### Scope (적용 범위)

NFT 전송에 적용됩니다. 받는 주소가 드레이너·제재 블록리스트에 등재된 경우입니다. (NftTransfer)

#### Audience (대상 사용자)

NFT를 보유하거나 전송하는 모든 사용자

#### Used Data (판정에 사용될 데이터)

받는 주소(`$.action.recipient`)와 체인 ID(`$.root.chain_id`)를 블록리스트에 조회한 결과(`recipientBlocklisted` — 블록리스트 등재 여부)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("nft-transfer-blocklisted-recipient-deny")
@severity("deny")
@reason("받는 주소가 알려진 드레이너/제재 블록리스트에 있습니다 — NFT 전송을 차단합니다. 주소를 다시 확인하세요.")
forbid(principal, action == Token::Action::"NftTransfer", resource)
when {
  context has custom
  && context.custom has recipientBlocklisted
  && context.custom.recipientBlocklisted
};
```
{% endcode %}
