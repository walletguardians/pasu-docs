# NFT-012: Seaport 주문의 만료가 지나치게 멀 시 경고

### Policy Definition (정책 정의)

> Seaport 주문의 유효 기간이 **6개월을 넘길** 만큼 길게 잡혀 있으면, 서명하기 전에 경고를 표시합니다.

Seaport에서 NFT를 내놓거나(리스팅) 오퍼를 넣으면, 그 주문에는 언제부터 언제까지 체결될 수 있는지가 적힌 유효 기간(`startTime`~`endTime`)이 함께 서명됩니다. 보통 리스팅은 며칠에서 몇 주 정도로 잡지만, 이 기간을 6개월보다 길게 잡으면 한 번 서명한 주문이 반년 넘게 살아 있게 됩니다. 오래 잠들어 있던 주문은 가격이 크게 움직인 한참 뒤에야 본인에게 불리한 값으로 체결될 수 있고, 잊고 지내는 사이 누군가 그 서명을 그대로 가져다 쓸 여지도 커집니다. 그래서 주문의 유효 기간이 6개월을 넘기면 서명 전에 한 번 더 확인하도록 알립니다.

#### Scope (적용 범위)

Seaport 주문 서명에 적용됩니다. 시작 시각이 0이 아니면서 `endTime`과 `startTime`의 차이가 15,552,000초(180일, OpenSea 리스팅 상한)를 넘을 때입니다. (Sign Order)

#### Audience (대상 사용자)

Seaport에서 NFT를 리스팅하거나 오퍼를 넣는 사용자

#### Used Data (판정에 사용될 데이터)

주문이 적용된 마켓플레이스(`context.venue.name`), 주문의 시작 시각(`context.startTime`)과 종료 시각(`context.endTime`)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("far-future-expiry-warn")
@severity("warn")
@reason("이 주문은 보통 리스팅보다 훨씬 오래 유효합니다 — 오래 살아 있는 서명 주문은 한참 뒤에 본인에게 불리하게 체결될 수 있으니 확인하세요")
forbid (
    principal,
    action == Marketplace::Action::"SignOrder",
    resource
)
when
{
    context.venue.name == "seaport" &&
    context.startTime != 0 &&
    (context.endTime - context.startTime) > 15552000
};
```
