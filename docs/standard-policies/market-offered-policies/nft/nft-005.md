# NFT-005: 구매한 NFT가 본인이 아닌 다른 주소로 갈 시 경고

### Policy Definition (정책 정의)

> Seaport 주문을 체결할 때 받을 **NFT가 서명자 본인이 아닌 다른 주소로** 가는 경우 경고를 표시합니다.

Seaport에서 주문을 체결(FulfillOrder)하면 대금은 내가 내지만, 받을 NFT가 어느 주소로 갈지는 주문 안에 따로 적혀 있습니다. 이 받는 주소가 내 지갑이 아니면, 결제는 내가 하고 NFT는 남의 지갑으로 넘어갑니다. 악성 사이트가 받는 주소만 바꿔치기한 주문을 내밀면 돈만 내고 NFT는 공격자에게 가는 구조입니다. 그래서 받는 주소가 서명자 본인 지갑과 다르면 체결 전에 경고를 표시합니다.

#### Scope (적용 범위)

Seaport 마켓플레이스 주문 체결에 적용됩니다. 받을 NFT의 수령 주소가 주문을 서명·제출하는 본인 지갑과 다를 때 발동합니다. (FulfillOrder — 구매 또는 오퍼 수락)

#### Audience (대상 사용자)

OpenSea 등 Seaport 기반 NFT 마켓플레이스에서 NFT를 구매하거나 오퍼를 수락하는 사용자

#### Used Data (판정에 사용될 데이터)

체결하려는 주문의 마켓플레이스(`context.venue.name` — `seaport`인지), NFT를 받을 주소(`context.recipient`), 주문을 제출하는 본인 지갑(`context.meta.submitter`). 두 주소가 다른지 비교합니다.

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("fulfill-recipient-not-self-warn")
@severity("warn")
@reason("이 구매로 받을 NFT가 본인 지갑이 아닌 다른 주소로 갑니다 — 승인 전에 받는 주소를 확인하세요")
forbid (
    principal,
    action == Marketplace::Action::"FulfillOrder",
    resource
)
when
{
    context.venue.name == "seaport" &&
    context.recipient != context.meta.submitter
};
```
{% endcode %}
