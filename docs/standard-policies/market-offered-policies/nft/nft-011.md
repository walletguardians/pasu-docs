# NFT-011: 비정상 타입의 마켓플레이스 주문에 서명할 시 차단

### Policy Definition (정책 정의)

> Seaport 주문이 정상 리스팅·오퍼가 아닌 **비정상 타입**(contract·unknown)인 경우, 서명이 나가지 않도록 **차단**합니다.

Seaport에서 NFT를 팔거나 사겠다고 내놓을 때는 주문에 타입이 붙습니다. 사람이 마켓플레이스에서 리스팅이나 오퍼를 만들면 일반(full·partial) 타입이 됩니다. 그런데 `contract` 타입은 외부 컨트랙트가 체결 조건을 마음대로 정하도록 넘기는 주문이고, `unknown`은 디코더가 어떤 주문인지 끝내 알아내지 못한 경우입니다. 둘 다 사람이 직접 리스팅·오퍼를 만드는 정상 흐름에서는 나오지 않습니다. 드레이너가 피해자에게 엉뚱한 서명을 받아내려 할 때 이런 형태가 보이므로, 이 타입의 주문 서명은 막습니다.

#### Scope (적용 범위)

Seaport 마켓플레이스 주문 서명에 적용됩니다. 주문 타입이 `contract` 또는 `unknown`일 때만 발동합니다. (Sign Order)

#### Audience (대상 사용자)

NFT를 사고파는 모든 사용자

#### Used Data (판정에 사용될 데이터)

서명하려는 주문의 마켓플레이스 이름(`context.venue.name` — seaport인지)과 주문 타입(`context.orderType` — contract·unknown 여부)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("contract-order-anomaly-deny")
@severity("deny")
@reason("이 주문 타입은 정상 마켓플레이스 리스팅·오퍼가 아닙니다(contract·unknown 타입) — 사람의 정상 서명 흐름에 없는 형태라 차단했습니다")
forbid (
    principal,
    action == Marketplace::Action::"SignOrder",
    resource
)
when
{
    context.venue.name == "seaport" &&
    ["contract", "unknown"].contains(context.orderType)
};
```
{% endcode %}

```json
}
```
