# AMM-010: 슬리피지 허용치가 1%를 초과하는 스왑을 실행할 시 경고

### Policy Definition (정책 정의)

> 스왑에 걸린 **슬리피지 허용치가 1%를 넘는** 경우, 샌드위치 공격에 손해 볼 수 있어 경고합니다.

스왑을 보낼 때는 '최소한 이만큼은 받겠다'는 하한선을 함께 정합니다. 이 하한선이 기대 수량에서 얼마나 벌어져 있는지가 슬리피지 허용치인데, 이 폭이 넓을수록 그만큼 적게 받고도 거래가 체결됩니다. 그 빈틈을 노려 봇이 내 거래 앞뒤로 끼어들어 가격을 밀었다 되돌리는 샌드위치 공격이 들어오면, 허용치만큼 손해가 고스란히 내 몫이 됩니다. 그래서 슬리피지 허용치가 1%(100bp)를 넘으면, 폭을 좁힐 수 있는지 서명 전에 확인하도록 경고합니다.

#### Scope (적용 범위)

DEX 스왑. 스왑에 설정된 슬리피지 허용치가 1%(100bp)를 초과하는 경우에 적용됩니다. (Swap)

#### Audience (대상 사용자)

거래량이 적은 토큰이나 변동성 큰 토큰을 자주 스왑하는 사용자

#### Used Data (판정에 사용될 데이터)

스왑 주문에 설정된 슬리피지 허용치 (`context.slippageBp` — 100bp = 1%)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("swap-high-slippage-warn")
@severity("warn")
@reason("슬리피지 허용치가 1%를 넘습니다 — 샌드위치 공격으로 손해 볼 수 있습니다")
forbid(principal, action == Amm::Action::"Swap", resource)
when { context.slippageBp > 100 };
```
{% endcode %}
