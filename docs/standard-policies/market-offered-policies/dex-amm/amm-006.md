# AMM-006: 거래량이 적은 풀에 유동성을 공급할 시 경고

### Policy Definition (정책 정의)

> 유동성을 넣으려는 풀의 **최근 24시간 거래량이 너무 적은** 경우, 공급 전에 다시 확인하도록 경고합니다.

유동성 공급은 풀에 토큰을 맡기고 그 대가로 LP 토큰을 받는 일입니다. 그런데 거래가 거의 없는 풀은 막상 자금을 빼려 할 때 받아줄 거래 상대가 없어 빠져나오기 어렵고, 누군가 먼저 적은 유동성을 조작해 LP 지분 가치를 부풀리는 share-inflation 공격이나 비영구적 손실(IL)에 그대로 노출됩니다. 풀의 24시간 거래량이 기준에 못 미치면, 정말 이 풀에 자금을 넣을지 서명 전에 한 번 더 따져보도록 경고합니다.

#### Scope (적용 범위)

DEX 유동성 공급(Add Liquidity). 공급하려는 풀의 최근 24시간 거래량이 미화 $10,000에 못 미치는 경우에 적용됩니다.

#### Audience (대상 사용자)

신생 풀이나 잘 알려지지 않은 풀에 LP로 들어가는 사용자

#### Used Data (판정에 사용될 데이터)

해당 풀의 최근 24시간 거래량 (`poolVol24hUsd` — 풀 유동성 조회 결과)

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("addliquidity-low-liquidity-warn")
@severity("warn")
@reason("이 풀은 24시간 거래량이 매우 적습니다 — LP를 빠져나오기 어렵고 share-inflation·비영구적 손실(IL)에 노출될 수 있습니다")
forbid(principal, action == Amm::Action::"AddLiquidity", resource)
when {
  context has custom
  && context.custom has poolVol24hUsd
  && context.custom.poolVol24hUsd.lessThan(decimal("10000.0000"))
};
```
