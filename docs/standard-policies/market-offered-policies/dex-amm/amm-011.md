# AMM-011: 낼 금액에 상한이 없는 스왑을 서명할 시 경고

### Policy Definition (정책 정의)

> 받을 토큰 수량만 고정하고 낼 입력 토큰에는 상한이 없는(무제한 max-in) **exact-output 스왑**일 때 경고합니다.

exact-output에서는 낼 입력 토큰의 양이 시세에 따라 달라지는데, 보통은 '최대 이만큼까지만 낸다'는 max-in 상한을 함께 걸어 둡니다. 이 상한이 uint256 최댓값(무제한)으로 설정되면 낼 금액에 사실상 천장이 없어집니다. 풀 가격이 불리하게 움직이거나 거래가 샌드위치 공격을 당하면 받을 토큰은 그대로지만 낸 입력 토큰은 예상보다 훨씬 많이 빠져나갈 수 있습니다. 따라서 max-in이 무제한으로 잡힌 exact-output 스왑은 서명 전에 낼 금액을 확인하도록 경고합니다.

#### Scope (적용 범위)

DEX 스왑(Swap). 입력 토큰의 상한(max-in)이 uint256 최댓값으로 설정된 경우에 적용됩니다.

#### Audience (대상 사용자)

Uniswap 등에서 exact-output 스왑을 쓰는 사용자

#### Used Data (판정에 사용될 데이터)

* 스왑 방식
* 입력 토큰의 상한값

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("swap-exact-output-unbounded-input-warn")
@severity("warn")
@reason("이 스왑은 지불할 입력 수량에 상한이 없습니다(무제한 max-in) — 지불 금액을 확인하세요")
forbid(principal, action == Amm::Action::"Swap", resource)
when {
  context.direction.kind == "exact_output"
  && context.direction has maxAmountIn
  && context.direction.maxAmountIn == "0xffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffff"
};
```
