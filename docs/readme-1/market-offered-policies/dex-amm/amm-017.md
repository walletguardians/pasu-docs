# AMM-017: 인텐트 주문으로 받을 토큰이 제3자에게 갈 시 경고

### Policy Definition (정책 정의)

> 인텐트 기반 거래(UniswapX·CowSwap 등)에서 토큰을 **받을 주소가 제3자**로 되어 있는 경우, 서명 전에 다시 확인하도록 경고합니다.

인텐트 거래는 거래소로 직접 보내는 대신, '이런 조건으로 바꿔 달라'는 주문서에 서명하면 solver가 대신 체결해 주는 방식입니다. 이때 결과물을 받을 주소가 내 주소가 아니면, 체결된 토큰이 그대로 남의 지갑으로 들어갑니다. 받을 주소가 본인 것이 아니면, 주소가 맞는지 서명 전에 확인하도록 경고합니다.

#### Scope (적용 범위)

인텐트 주문 서명 시 받을 주소가 서명자 본인이 아닌 경우에 적용됩니다.

#### Audience (대상 사용자)

인텐트 기반 거래를 사용하는 사용자

#### Used Data (판정에 사용될 데이터)

서명자 본인 주소와 주문의 수령 주소(`recipient)` 가 동일한지 확인합니다.

#### Policy in Code

{% code title="policy.cedar" %}
```solidity
@id("intent-order-recipient-not-self-warn")
@severity("warn")
@reason("수령 토큰이 본인 지갑이 아닌 제3자에게 전달됩니다 - 수령 주소를 확인하세요.")
forbid(principal, action == Amm::Action::"SignIntentOrder", resource)
when {
  context.recipient != principal.address
};
```
