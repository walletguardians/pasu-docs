# 마켓플레이스에 배포하기

직접 만든 정책이나 세트를 마켓플레이스에 올려 다른 사용자와 공유할 수 있습니다.

## 게시 전 준비

1. **정책 완성** — 에디터에서 조건·값·강도 작성 ([에디터](editor.md))
2. **검증** — [시뮬레이션](../user-guide/simulation.md)으로 의도한 판정이 나오는지 확인
3. **문서 작성** — 정의 / 범위 / 대상 / 데이터 탭 채우기 (한국어·영어 권장)

## 게시 흐름

에디터의 **게시(Publish)** → 게시 모달에서:

* **빈칸(hole) 처리** — 사용자가 설치 시 채울 파라미터를 구멍으로 비워둠 (예: 금액 기준)
* **미리보기** — 게시될 모습 확인 (세트는 포함 정책 목록·카테고리 태그)
* **버전** — 초기 게시는 `1.0.0`. 이후 SemVer로 올림
* **변경 로그(changelog)** — 버전별 변경 내용

게시하면 기본 게시자 등급은 **community** 입니다.

## 정책 vs 세트

| 종류 | 게시 내용 |
|------|-----------|
| **정책(policy)** | Cedar 텍스트 1개 |
| **세트(set)** | 여러 정책을 묶음 (멤버를 스냅샷으로 내장) |

## 메타데이터

게시 시 함께 넣는 정보:

* `slug` (고유, 대소문자 구분), 표시 이름(한/영), 설명(한/영)
* `domain` (예: `uniswap`, `aave`), `category` (예: 승인, 스왑, perps)
* `severity`, `doc`(정의/범위/대상/데이터), `intents`

## 게시자 등급(tier)

| 등급 | 의미 |
|------|------|
| **Official** | DAMBI 팀 공식 |
| **Verified** | 검증된 3자 |
| **Community** | 누구나 (기본값) |

> 등급 승격은 별도(out-of-band) 절차로 이뤄집니다.

## 버전 관리

* **새 버전** — 같은 정책에 SemVer로 새 버전 게시 (현재보다 반드시 높아야 함)
* 사용자는 설치 시점의 버전을 받고, 그 버전에 리뷰를 남깁니다

## 내 게시물 관리

프로필 → **게시한 정책(Published)** 에서:

* 정책/세트별 설치수·버전·상태(pending/published/archived/rejected) 확인
* 새 버전 게시, 삭제

## 모더레이션 & 안전

* 사용자는 위험·오해 소지·스팸·악용 등으로 정책을 **신고**할 수 있습니다.
* 신고는 모더레이션 큐로 전달되어 관리자가 상태(open/resolved)를 처리합니다.
* 안전한 정책을 위해, 게시 전 시뮬레이션 검증과 명확한 문서 작성을 권장합니다.
