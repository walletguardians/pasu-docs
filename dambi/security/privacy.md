DAMBI의 설계 원칙은 **"판정은 기기 안에서, 데이터는 최소한으로"** 입니다. 트랜잭션/서명 평가는 확장프로그램 내부 WASM에서 끝나며, 평가 자체에는 서버 전송이 필요하지 않습니다.

> 기술적인 데이터 흐름 전체는 [데이터 흐름](data-flows.md), 신뢰·서명 모델은 [보안 모델](security-model.md)을 참고하세요. 이 페이지는 그중 공개용 처리방침에 해당합니다.

## 1. 우리가 다루는 데이터

### 기기에만 저장되는 데이터 (서버로 전송하지 않음)

| 데이터 | 설명 | 위치 |
|--------|------|------|
| **설치한 정책** | 직접 만들거나 정책 허브에서 받은 정책 | 확장 로컬 저장소 (`ps2:*`) |
| **판정 히스토리** | 가로챈 트랜잭션/서명과 그 판정·이유, 사용자 결정 | 확장 로컬 저장소 |
| **설정** | 서버 주소, 테마 등 | `localStorage` / `chrome.storage` |

> 트랜잭션/서명 평가는 기기 안 WASM에서 처리됩니다. 평가만 사용하는 한, 트랜잭션 내용은 서버로 나가지 않습니다.

### 백엔드(policy-server)와 연결될 때 처리되는 데이터

로그인하거나 정책 허브/동기화/예측 기능을 쓰면 다음이 서버와 오갈 수 있습니다.

| 데이터 | 언제 | 목적 |
|--------|------|------|
| **계정 정보(이메일 등)** | Google 로그인 시 | 인증·세션(JWT) |
| **지갑 주소 및 상태** | 지갑 등록·상태 동기화 시 | 잔고/상태 컨텍스트, 예측 델타 |
| **평가용 remote facts 요청** | 정책이 외부 사실(예: USD 가치)을 요구할 때 | 해당 사실 조회 (`policy-rpc`) |
| **정책 허브 활동** | 정책 게시·리뷰·평점·신고 시 | 정책 허브 운영 |

### LLM 보조 정책 초안 (선택 기능)

대시보드에는 정책 초안을 LLM으로 생성하는 보조 기능이 있습니다. 이 기능을 사용하면 입력한 내용이 LLM 제공자에게 전송될 수 있습니다.
