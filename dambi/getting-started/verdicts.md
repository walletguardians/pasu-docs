# 판정 이해하기 (Pass / Warn / Fail)

DAMBI가 행동을 평가하면 결과는 항상 셋 중 하나입니다. 이 **판정(Verdict)** 이 DAMBI 사용 경험의 핵심입니다.

<사진 첨부 필요>&#x20;

| 판정          | 의미                          | 사용자 경험                       |
| ----------- | --------------------------- | ---------------------------- |
| 🟢 **Pass** | 설치된 정책 중 어디에도 걸리지 않음        | 별도 개입 없이 그대로 진행              |
| 🟡 **Warn** | 위험 신호가 있음. 막지는 않되 사용자 확인 필요 | 경고 화면 표시 → 사용자가 **계속/취소** 선택 |
| 🔴 **Fail** | 정책이 명시적으로 차단                | 서명/전송이 막힘                    |

> 판정은 **기기 안의 WASM Cedar 엔진**이 내립니다. 네트워크 왕복 없이 로컬에서 결정되며, 최종 판정의 주체는 언제나 확장프로그램입니다.

***

## 🟢 Pass (통과)

행동이 설치된 어떤 정책에도 걸리지 않았다는 뜻입니다. 평소 흐름을 방해하지 않도록, Pass는 보통 별도 화면 없이 지나갑니다.

> ⚠️ **"Pass = 100% 안전"은 아닙니다.** "내가 설치한 규칙들엔 안 걸렸다"는 의미입니다. 보호 범위는 설치한 정책에 달려 있으니, 자주 쓰는 도메인(토큰·DEX·NFT 등)의 정책을 갖춰두는 게 좋습니다.

## 🟡 Warn (경고)

위험 가능성이 있지만 정상적인 사용일 수도 있는 행동입니다. DAMBI는 막지 않고 **판단을 사용자에게 넘깁니다.**

* 예: 기준 금액을 넘는 스왑, 평소보다 큰 토큰 승인, 높은 레버리지 주문
* 확인 화면에 **이유**(어떤 정책·규칙이 걸렸는지)가 표시됩니다.
* 사용자는 **계속 진행** 또는 **취소** 를 선택하고, 그 결정은 [히스토리](verdicts.md#히스토리에서-다시-보기)에 남습니다.

## 🔴 Fail (차단)

정책이 명백히 위험으로 판단해 막는 경우입니다.

* 예: 제재 주소로의 전송, 무제한/위험 스펜더 승인, 알려진 악성 패턴
* 해당 서명/트랜잭션은 차단됩니다.

### 안전을 위한 차단 (fail-closed)

정상 정책 위반과 별개로, DAMBI는 **확실하게 판단할 수 없을 때 안전한 쪽으로 닫습니다.**

* 어떤 어댑터로도 **해석되지 않는** 요청 → 알 수 없는 행동(`Unknown`)으로 처리
* 엔진 오류 등으로 평가가 불가능한 경우

이때의 판정 출처는 `fail_closed`로 표시됩니다(정상 평가의 출처는 `declarative-v2`). 이는 **레거시 표시가 아니라 의도된 안전 장치**입니다. "모르면 일단 경고/차단".

***

## 정책 강도: deny / warn / info

정책을 **작성**할 때는 규칙별로 강도를 정합니다. 이 강도가 위 판정으로 이어집니다.

| 작성 시 강도 | 결과 판정   | 용도            |
| ------- | ------- | ------------- |
| `deny`  | 🔴 Fail | 반드시 막아야 하는 위험 |
| `warn`  | 🟡 Warn | 확인이 필요한 위험    |
| `info`  | (정보성)   | 차단·경고 없이 알림만  |

> Cedar에서 효과(effect)는 항상 `forbid`이고, **강도(severity)로 차단/경고/정보를 구분**합니다.

***

## 히스토리에서 다시 보기

대시보드 **히스토리**에는 가로챈 모든 요청과 그 판정이 쌓입니다.

* Pass / Warn / Fail 건수 요약
* 각 건의 판정 이유(매칭된 정책·규칙)
* Warn에서 사용자가 내린 결정(진행/취소) 기록

이를 통해 "왜 막혔지?", "지난번에 내가 뭘 승인했지?"를 언제든 되짚어볼 수 있습니다.
