# 시뮬레이션으로 테스트하기

시뮬레이션은 실제 dApp에 가기 전에, 정책이 의도대로 동작하는지 **안전하게(자금 이동 없이)** 확인하는 도구입니다. 실제 WASM 엔진으로 평가하므로 결과가 운영과 동일해요.

## 4단계 마법사

### 1단계 · 지갑 선택

* 등록된 지갑 목록에서 테스트할 지갑 체크 (전체 선택/해제)
* 체인 선택 (예: `eip155:1`)
* 각 지갑의 보유·승인·포지션이 미리 로드됩니다

### 2단계 · 정책 선택

* 지갑별로 적용된 정책 세트 표시
* 정책 on/off 토글 (지갑별 범위)로 어떤 규칙을 테스트할지 조정

### 3단계 · 트랜잭션 큐

* calldata 빌더로 테스트할 요청을 행 단위로 입력 (from / to / calldata / value)
* 행마다 보낸 지갑(fromWallet) 지정 (최대 99행)
* 각 행이 해당 지갑 상태를 기준으로 **디코드 → 평가 → 상태 반영** 체인을 거칩니다

### 4단계 · 결과

* 지갑별 상태 요약 (토큰·승인·포지션·대기·가스)
* **스크러버** | 초기 상태부터 N단계까지 상태 변화를 단계별로 추적
* 각 단계의 **판정과 매칭된 규칙**
* 차단(fail)된 경우 정책 구조 다이어그램과 **"어디서 막혔는지"** 진단

## 어떻게 읽나

* 기준을 넘는 행동을 넣었을 때 의도한 강도로 `Warn`/`Fail`이 떠야 정상입니다.
* `Pass`만 나온다면 정책이 안 걸리는 것이니 조건/값을 다시 확인하세요.
* 차단 진단을 보면 어떤 조건 때문에 막혔는지 알 수 있어, 정책 디버깅에 유용합니다.

## 히스토리에서 재시뮬레이션

[히스토리](history.md)의 각 기록에서 **재시뮬레이션** 링크를 누르면 그 트랜잭션 필드가 시뮬레이션으로 넘어와, "왜 이렇게 판정됐는지"를 그대로 재현할 수 있습니다.
