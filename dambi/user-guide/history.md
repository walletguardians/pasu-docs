# 히스토리 & 감사 로그

히스토리는 DAMBI가 가로챈 모든 트랜잭션·서명과 그 판정을 시간순으로 쌓아둔 기록입니다. "왜 막혔지?", "지난번에 뭘 승인했지?"를 되짚어볼 수 있어요.

> 이 기록은 **기기 로컬**(확장 저장소, 최대 1000건)에 보관됩니다. 서버로 전송되지 않아요. → [데이터 흐름](../security/data-flows.md)

## 한눈에 보기

상단에 **Pass / Warn / Fail 건수 요약**이 표시됩니다.

표의 각 행:

| 항목 | 설명 |
|------|------|
| 판정 | fail / warn / pass 표시 |
| 시간 | 초 단위 타임스탬프 |
| dApp·함수 | 요청 출처(origin)와 호출 함수 |
| 지갑 | 대상 지갑 |
| 정책 | 매칭된 정책 + 강도 |
| 이유(reason) | 왜 그렇게 판정됐는지 |
| 사용자 결정 | Warn에서 진행/취소 선택 결과 |

## 행 펼쳐보기

행을 클릭하면 상세 패널이 열립니다:

* 매칭된 정책 이름 + 강도
* RPC 메서드 / 디코드된 함수 이름
* 대상 컨트랙트 주소 + 심볼
* selector 시그니처 / 디코드 이름
* 지갑·dApp origin·전체 타임스탬프·전체 이유 텍스트
* **상태 변화(state diff)** — 토큰/포지션/대기 변화 + 가스
* **재시뮬레이션** 링크 — 이 트랜잭션을 시뮬레이션으로 재현 ([시뮬레이션](simulation.md))
* 차단(deny) 행: 정책 구조 다이어그램 + "어디서 막혔는지" 진단

## 그룹·필터

**그룹 보기**
* 시간(time) / 활동(activity, 같은 지갑·origin 연속 묶기) / 판정(verdict) / origin / 규칙(rule)

**필터**
* 기간(range): 1시간 / 6시간 / 24시간 / 전체
* 검색: 지갑·origin·함수·메서드·정책·이유·컨트랙트·selector
* 판정 토글: fail / warn / pass

## 미해결 경고 배지

팝업 아이콘에는 아직 결정하지 않은 경고(unresolved warn) 개수가 배지로 표시됩니다(히스토리 화면에서 약 30초 주기로 갱신).
